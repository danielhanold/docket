## Findings

**1. ADR-0026 — the decision this build was really making.** Recorded on `origin/docket` as `0026-fork-dispatch-opacity-two-invocation-paths.md`: fork-dispatch's TUI opacity is **accepted, not fixed** — no log-tailer, no progress protocol — and docket instead documents two first-class invocation paths into the same pinned wrapper, naming the on-disk transcript as the escape hatch. Observability becomes a choice the caller makes at invocation time. Its `## Context` carries the five 2026-07-12 / CC 2.1.207 findings, including the one that closes ADR-0024's open question: **the self-preloading cycle is safe** (a wrapper whose `skills:` preloads the very skill that forks into it does not recurse — preload is startup content injection, the fork fires on invocation).

**2. `adrs: [24, 26]` is load-bearing — do not "tidy" 24 out of it.** `terminal-publish` copies the *current bytes* of every `Accepted` ADR named in `adrs:`. ADR-0024's new forward-pointer `## Update` note lives only on `docket` right now; listing **24** is the only thing that carries it onto `main` at merge. Drop it and `main`'s copy of 0024 stays stale forever — a reader there would never learn its open question was closed, which is the exact failure the note exists to prevent.

**3. The teaching prose was factually false on the first pass — caught by the whole-branch review, not by the suite.** The worked illustration claimed a design pass "sits between" the build and the sweep, and that a loop reads *groom at the mid tier → build at the top tier → sweep at the cheap tier*. Against the shipped built-ins that is wrong: `docket-auto-groom` runs at **opus/xhigh — the same top tier as `docket-implement-next`**, and the only pinned mid-tier agents are `docket-adr` and `docket-finalize-change` (sonnet/medium). `docket-status` was also called "low effort" when it ships at `medium`. Corrected to *design and build at the top tier → record the ADR at the mid tier → sweep at the cheap tier*, which is both true and the better lesson (designing the work is no cheaper than doing it). **The sentinels could never have caught this** — they pin the *framing*, not the *claims*. Worth remembering: a doc sentinel proves the sentence still exists, never that it is still true.

**4. Two doc sentinels shipped double-guarded and had to be re-anchored** (both caught in task review, both fixed):
- `skill-invoke`/`agent-dispatch` appear in *both* the comparison table and the rule-of-thumb sentence below it, so deleting the entire table left the assert **green**. Re-anchored to the table-unique literals `| **Skill-invoke** |` / `| **Agent-dispatch** |`.
- `"cheap tier"` appears in the *What you get* bullet that the same task added, so that clause had **zero** discriminating power. Re-anchored to the block-exclusive thesis, `matched to the …session… instead of`.

Both are fresh instances of LEARNINGS #21/#36-37 — an assert must own exactly one clause, and mutation-testing it in isolation is what exposes the ones that do not. In both cases the *first* mutation test surfaced the defect and the implementer's report rationalized it away; the reviewer re-deriving the occurrence counts by hand is what actually caught them.

**5. Plan deviation (adjudicated against the spec).** The plan prescribed a ~10-line paragraph for `references/agent-layer.md`. Review found it re-teaching README's table, transcript path, version stamp, and restart caveat near-verbatim — the duplication the "three homes, three altitudes" split exists to prevent. Adjudicated **against the spec, which governs the plan**: §4 explicitly requires this file to carry both-paths-same-wrapper, the observability distinction, *and* the process-start caveat, but "mechanics only, ~6 lines… the teaching and the table stay in README." So the paragraph was compressed to cross-reference README rather than restate it, keeping the two genuine *mechanics* (the no-recursion composition rule and process-start registration). The caveat was kept, not deleted — removing it would have contradicted the spec.

## Follow-ups

- **The tier prose is now a drift surface.** Finding #3 exists because README prose asserts what tier each agent runs at while the truth lives in `agents/docket-*.md`. Spec A7 avoided *literal model ids* for exactly this reason, but the *tier words* (cheap/mid/top) drift just as silently — an `agents:` retune (e.g. moving `docket-adr` to haiku) would falsify the README with nothing to catch it. Worth a small change: a sentinel that derives each built-in's tier from `agents/docket-*.md` and asserts the README's illustration still agrees, or a decision to stop naming per-agent tiers in prose at all.
