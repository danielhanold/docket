## Findings

- **Two Important test-robustness defects, found in task review and fixed (commit `b41150e`).** (1) The `assert()` helper ran `eval "$2"` in the current shell, so an assert body containing `for … || exit 1` would terminate the whole test on failure instead of recording one `NOT OK` and continuing — fixed by wrapping the eval in a subshell `( eval "$2" )` (`fail=1` still propagates from the parent-scope else branch). (2) The whole-backlog `/loop` drain assert regex `/loop docket-implement-next$|/loop docket-implement-next[^0-9]` also matched the id-set bullet (the space after `next` is a non-digit), so it gave no independent regression protection — retargeted to a fixed string requiring the trailing backtick right after `next`. A Minor "never merges" assert that matched a pre-existing Quickstart sentence was also scoped to the new subsection's phrasing.
- **Metadata-branch note (learning `metadata-branch-invisible-to-suite`).** The repo test suite sees only the integration branch. The disposition/id-set vocabulary is guarded by `tests/test_loop_continuation.sh` (which ships on the feature branch and merges with the code) and paired with the whole-branch review; verified green at build time.
- **ADR judgment: no new ADR.** The four-disposition contract is a design-time decision already captured comprehensively in the spec (§3.1), not a fresh non-obvious implementation-time decision; the §6 degrade was pre-authorized by the spec. With `terminal_publish: true`, the spec is published to `main` at close-out, so the contract's rationale is preserved permanently on the code line without a restating ADR. The change cites ADR-0001 (the CAS-claim model it builds on).
- **Size budget raised in-diff.** `skills/docket-implement-next/SKILL.md` grew from 108/2228 to 127/2597 (lines/words); the `tests/test_skill_size_budgets.sh` row was raised `119 2451 → 140 2845` (post-edit actuals + ~10%), which the guard explicitly permits.
