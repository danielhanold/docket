# A `bootstrap` facade verb — results
Change: #74 · Branch: feat/bootstrap-facade-verb · PR: (opened at close-out) · Plan: docs/superpowers/plans/2026-07-14-bootstrap-facade-verb.md · ADRs: 29, 30

Routing/surface change: `docket.sh bootstrap` routes the `CREATE_ORPHAN` fresh-repo path through the facade; the convention's Step-0 stops naming a second binary, and the 0072 wiring guard flips from tolerating the `docket-config.sh --bootstrap` carve-out to forbidding it. `scripts/docket-config.sh` behavior is untouched.

## Verify (human)

- [ ] (Optional smoke — the CREATE_ORPHAN path is human-attended by definition.) On a genuinely fresh repo (a `main` with no `docket` branch and no `docs/changes/active` planning surface), `docket.sh bootstrap` creates + pushes the orphan `docket` branch, seeds the managed `.gitignore` block, and a subsequent `docket.sh preflight` verdicts `PROCEED`. Covered by `tests/test_docket_facade.sh (F)` against the real resolver, but this is the one path that only truly exercises on a first-run repo.

## Findings

- **Spec §5 exit-code claim was wrong — corrected in the test (LEARNINGS #21).** The spec said `docket.sh bootstrap` in a `STOP_MIGRATE`-shaped repo "exits non-zero and writes nothing." Verified empirically (2026-07-14): it exits **0**, emits `BOOTSTRAP=STOP_MIGRATE`, and writes nothing. The resolver *reports* the verdict and exits 0 — failing closed on a non-`PROCEED` verdict is `preflight`'s job, not the verb's. Since changing `docket-config.sh` behavior is out of scope, the functional test asserts the true behavior (exit 0 + `STOP_MIGRATE` + no write). The spec's defensible *conclusion* (the cell guard holds through the facade — no write outside `CREATE_ORPHAN`) is preserved; only its exit-code detail was corrected. Not silently patched — recorded here.
- **`bootstrap` emits the resolver's default `shell` format, not the model-facing plain block.** `env`/`preflight` force `--format plain`; `bootstrap` passes only `--bootstrap`, so its stdout is `%q`-quoted (`FORMAT=shell` default). This is by design (the verb is pure routing, not a composite): the Step-0 flow is `bootstrap` (create the orphan) then `preflight` (sync + print the plain block the agent reads). `scripts/docket.md` documents this explicitly so no one parses `bootstrap`'s own stdout as literals.
- **Cross-cutting sentinel regression, caught by the whole-suite run, not the plan (LEARNINGS enumerated-floor / whole-suite).** Retiring the carve-out removed the only *slash-prefixed* `/docket-config.sh` string from the convention, reddening a pre-existing sentinel in `tests/test_docket_config.sh` (`"convention names docket-config.sh"`, keyed on `/docket-config.sh`). The convention still NAMES the resolver descriptively (`docket-config.sh --export` in the Config section) — exactly what ADR-0030 permits — so the sentinel's intent survives; only its invocation-coupled pattern was stale. Narrowed `/docket-config.sh` → `docket-config.sh` (kept, not deleted — LEARNINGS: narrow an old absolutist sentinel to the load-bearing property). The full 38-file suite is green.
- **Reconcile folded in the #71 guard reshape.** Change #71 (PR #81) merged after this spec was groomed and added the Layer-3 board-surfaces sentinel to `tests/test_skill_facade_wiring.sh`, shifting the two 0074 edit sites. The spec's `~78`/`~110` line refs were updated to shape-anchored descriptions; the 0074 edits are independent of Layer 3 and coexist cleanly (whole suite green).
- **ADR-0030 takes a dated `## Update` note (metadata branch).** ADR-0030's Decision named "two byte-exact canonical forms … and the single `…/docket-config.sh --bootstrap` carve-out"; after 0074 the guard strips only `…/docket.sh <op>` and asserts zero prefixed `docket-config.sh` invocations. The invocation-prefix discriminator is unchanged, so this is an Update note (never a Decision edit). `adrs: [29, 30]` already lists it, so terminal-publish re-copies the updated ADR-0030 onto `main` at merge (LEARNINGS #17). Written on `metadata_branch`, never on this feature branch.
- **§7 — the "two command shapes" framing check (input to change 0073's groom).** Whole-skill-surface grep for any non-facade `${DOCKET_SCRIPTS_DIR`-prefixed invocation: **none remains** — `docket.sh <op>` is the only such invocation shape in skill prose. docket's agent-facing runtime surface (what a Bash permission config must allow) is now a single binary, `docket.sh`; the carve-out no longer forces the permission config / the 0073 guide to enumerate a second binary. (Human-tier setup scripts `install.sh`/`migrate-to-docket.sh` remain, but are human-invoked and are only NAMED descriptively in skill prose, never invoked.) Change 0074 does NOT edit change 0073's stub — this verdict is input to 0073's own groom.
